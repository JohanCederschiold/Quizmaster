På vilka två sätt kan man får ett repository?:Genom att själv initiera ett (git init) eller genom att ladda ner(clone) ett.
Vad är en tracked file?:Det är en fil som var med i senaste commiten. De kan vara modified, unmodified eller staged. 
Vad är en untracked file?:Det är en fil som git inte bryr sig om eller känner till. 
Vad är det enklaste sättet att kolla tillståndet på dina filer?:git status eller git status -s (för kortare version).
Hur får man git att börja tracka en fil?:Genom att staga filen med git add <filnamn>
Vad händer om man gör ändringar i en stagead fil efter att den är staged men innan man gjort en commit?:Om man kör git status kommer filen att stå både som staged och som modified.
Vad är globpatterns?: Det är förenklade regular expressions
Hur får man git att ignorera filer?:Genom att skapa en .gitignore fil som innehåller globpatterns eller filnamn som den skall ignorera.
Hur kan man stagea alla tracked files och commit med en en rad?:git commit -a -m <commit_message>
Vad gör git rm kommandot?:Det tar bort filen från working directory OCH meddelar git att den inte skall trackas mer.

Med vilket kommando kan man se sin commithistrik med grafiska grenar (branches)?:git log --graph
Vad gör -p eller --patch växeln på log kommandot?:Visar de faktiska kondändringarna från commit till commit.
Hur kan man se kort statistik om alla (eller vissa) commits?: Med git log --stat.
Hur kan man begränsa antalet utskrifter med git log?: Men antal git log -2 (två senaste), med --since, --after, --until, --before eller med via strängar eller författare.

Vad är origin?:Det är default namnet som git ger till en godtycklig remote. 
Hur kan jag få git att visa med de URL:er som är förknippade med mina remotes?:git remote -v 
Hur lägger man till en ny remote?:Med git remote add<shortname> <url>.
Vad gör git fetch <remote>?:Det laddar ner ändringar från servern utan att merga dem med min version.
Vad gör git pull?: Kommandot hämtar OCH mergar med min version (om möjligt). Det motsvara alltså git fetch<remote> och git merge<remote>.
Hur skickar jag mina förändringar till en remote?:Med git push eller git push <remote><branch>

Vilka två tagtyper kan förekomma?:Lightweight och annotated?

Hur skapar man en lightweight tag?: git tag <namn> OBS inga andra växlar.

Hur överför jag mina tags till en remote server?: git push origin <tagname>
